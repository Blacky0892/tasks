# Система задач

Минималистичное веб-приложение для совместного ведения списков покупок, домашних задач и других семейных дел.

Проект сделан с акцентом на простоту: без сложной регистрации, без лишних экранов и с удобным интерфейсом для ежедневного использования с телефона или компьютера.

## Возможности

* создание нескольких списков;
* добавление задач в выбранный список;
* отметка задач как выполненных;
* автоматическое перемещение выполненных задач вниз;
* отображение автора задачи;
* отображение пользователя, который выполнил задачу;
* простая авторизация по персональной ссылке;
* аккуратный адаптивный интерфейс;
* поддержка аватарок пользователей.

## Идея проекта

Приложение предназначено для небольшого круга пользователей, например для семьи.

Каждому пользователю можно выдать личную ссылку для входа. По этой ссылке система понимает, кто именно открыл приложение, и показывает действия от его имени.

Например:

* Антон добавил задачу «Купить молоко»;
* Мария отметила её выполненной;
* задача переместилась вниз списка.

## Технологии

Проект использует:

* Laravel;
* Vue 3;
* Vite;
* Tailwind CSS;
* MySQL / MariaDB.

## Основные сущности

### Пользователь

Пользователь — участник системы, который может добавлять и выполнять задачи.

У пользователя есть:

* имя;
* аватарка;
* уникальный токен для входа по ссылке.

### Список

Список объединяет задачи по смыслу.

Примеры списков:

* Покупки;
* Дом;
* Дача;
* Планы;
* Документы.

### Задача

Задача — отдельный пункт внутри списка.

У задачи есть:

* текст;
* список, к которому она относится;
* автор;
* статус выполнения;
* пользователь, который отметил задачу выполненной;
* дата создания;
* дата выполнения.

## Авторизация

Авторизация работает по персональной ссылке.

Пользователю не нужно вводить логин или пароль. Достаточно открыть свою ссылку, например:

```text
https://example.com/login/unique-token
```

После перехода по ссылке система определяет пользователя и сохраняет его в сессии.

Такой вариант удобен для домашнего приложения, где не нужна сложная регистрация, но важно понимать, кто именно выполняет действия.

## Установка проекта

Склонировать репозиторий:

```bash
git clone git@github.com:username/repository.git
cd repository
```

Установить PHP-зависимости:

```bash
composer install
```

Установить frontend-зависимости:

```bash
npm install
```

Создать файл окружения:

```bash
cp .env.example .env
```

Сгенерировать ключ приложения:

```bash
php artisan key:generate
```

Настроить подключение к базе данных в `.env`:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=tasks
DB_USERNAME=root
DB_PASSWORD=
```

Выполнить миграции:

```bash
php artisan migrate
```

Запустить backend:

```bash
php artisan serve
```

Запустить frontend-сборку для разработки:

```bash
npm run dev
```

## Сборка для production

Для production-сборки выполнить:

```bash
npm run build
```

После этого собранные frontend-файлы будут находиться в директории `public/build`.

## Возможный сценарий использования

1. Администратор создаёт пользователей.
2. Для каждого пользователя генерируется персональная ссылка.
3. Пользователь открывает приложение по своей ссылке.
4. Создаёт список или выбирает уже существующий.
5. Добавляет задачи.
6. Другой пользователь видит задачи у себя.
7. После выполнения задача отмечается галочкой и перемещается вниз.

## Безопасность

Приложение не использует классическую авторизацию по логину и паролю.

Вместо этого используется вход по уникальному токену. Поэтому важно:

* не публиковать персональные ссылки в открытом доступе;
* использовать достаточно длинные случайные токены;
* при необходимости пересоздавать токен пользователя;
* включить HTTPS на production-сервере.

## Деплой

Типовой порядок деплоя:

```bash
composer install --no-dev --optimize-autoloader
npm ci
npm run build
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

Для Laravel-приложения важно, чтобы публичная директория сайта указывала на папку:

```text
public
```

Если хостинг использует `public_html`, можно настроить симлинк или скопировать содержимое публичной директории в нужное место в зависимости от возможностей хостинга.
