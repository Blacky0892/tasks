# Наш дом — семейные списки задач

Минималистичное веб-приложение для совместного ведения покупок, домашних дел и других семейных списков.

Проект рассчитан на небольшой круг доверенных пользователей: вход выполняется по персональной magic-ссылке, а интерфейс оптимизирован для ежедневного использования с телефона. Приложение можно установить как PWA, открывать ранее загруженные списки без сети и добавлять задачи в офлайн-очередь с последующей синхронизацией.

## Возможности

* создание нескольких семейных списков с эмодзи-иконками;
* редактирование названия и иконки списка;
* архивирование списков без физического удаления данных;
* ручная сортировка списков и активных задач;
* добавление одной или нескольких задач за раз;
* отметка задач как выполненных и возврат в активные;
* отдельный блок выполненных задач;
* повтор выполненных задач в списке;
* отображение автора задачи и пользователя, который её выполнил;
* цветные пользовательские аватарки;
* персональный вход по magic-ссылке без пароля;
* PWA-режим: установка на устройство, manifest, service worker;
* офлайн-страница и кеширование открытых экранов;
* локальная очередь задач, добавленных без сети;
* периодическая проверка изменений и мягкая синхронизация интерфейса.

## Как это работает

Приложение предназначено для семьи или другой небольшой группы пользователей.

1. Администратор создаёт пользователя через artisan-команду.
2. Команда генерирует персональную ссылку вида `https://example.com/magic/<token>`.
3. Пользователь открывает ссылку один раз, приложение определяет его и сохраняет авторизацию в сессии.
4. Все дальнейшие действия — создание списков, добавление и выполнение задач — сохраняются от имени этого пользователя.

Пример сценария:

* Антон добавляет в список «Покупки» задачу «Купить молоко»;
* Марина отмечает задачу выполненной;
* задача перемещается в блок выполненных, а рядом отображается, кто её закрыл.

## Установка

Склонируйте репозиторий и перейдите в директорию проекта:

```bash
git clone git@github.com:username/repository.git
cd repository
```

Установите backend-зависимости:

```bash
composer install
```

Установите frontend-зависимости:

```bash
npm install
```

Создайте файл окружения и ключ приложения:

```bash
cp .env.example .env
php artisan key:generate
```
Настройте подключение к базе данных в `.env`:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=tasks
DB_USERNAME=root
DB_PASSWORD=
```

Выполните миграции:

```bash
php artisan migrate
```

При необходимости можно использовать готовый composer-скрипт первоначальной настройки:

```bash
composer run setup
```

> Скрипт установит зависимости, создаст `.env`, сгенерирует ключ, выполнит миграции и соберёт frontend.

## Создание пользователей

Создать пользователя с magic-ссылкой можно командой:

```bash
php artisan family:create-user "Антон"
```

Можно указать email явно:

```bash
php artisan family:create-user "Мария" maria@example.com
```

После выполнения команда выведет персональную ссылку:

```text
https://example.com/magic/<token>
```

Передайте её пользователю приватно. Токен хранится в базе только в виде SHA-256-хеша.

## Основные маршруты

| Метод    | Путь                          | Назначение                     |
|----------|-------------------------------|--------------------------------|
| `GET`    | `/magic/{token}`              | вход по персональной ссылке    |
| `GET`    | `/`                           | главная страница со списками   |
| `POST`   | `/lists`                      | создание списка                |
| `GET`    | `/lists/{list}`               | страница списка                |
| `PATCH`  | `/lists/reorder`              | сортировка списков             |
| `PATCH`  | `/lists/{list}`               | редактирование списка          |
| `DELETE` | `/lists/{list}`               | архивирование списка           |
| `POST`   | `/lists/{list}/tasks`         | создание задач                 |
| `PATCH`  | `/lists/{list}/tasks/reorder` | сортировка активных задач      |
| `POST`   | `/tasks/{task}/toggle`        | переключение выполнения задачи |
| `PATCH`  | `/tasks/{task}`               | редактирование задачи          |
| `DELETE` | `/tasks/{task}`               | удаление задачи                |
| `GET`    | `/offline`                    | fallback-страница без сети     |

## Основные сущности

### Пользователь

Участник системы, который может создавать списки, добавлять задачи и отмечать их выполненными.

Ключевые поля:

* имя;
* email;
* хеш magic-токена;
* дата последнего входа;
* цвет аватарки.

### Список

Группа задач по смыслу: покупки, дом, дача, документы, планы.

Ключевые поля:

* название;
* эмодзи;
* порядок сортировки;
* автор;
* дата архивирования.

### Задача

Отдельный пункт внутри списка.

Ключевые поля:

* текст;
* заметка;
* список;
* порядок сортировки;
* статус выполнения;
* автор;
* пользователь, который выполнил задачу;
* дата выполнения.

## PWA и офлайн-режим

Приложение содержит `manifest.webmanifest` и service worker.

Что поддерживается:

* установка приложения на домашний экран;
* кеширование статических ресурсов и ранее открытых страниц;
* отдельная страница `/offline` при отсутствии сети;
* уведомление о доступном обновлении service worker;
* добавление задач без сети в локальную очередь `localStorage`;
* автоматическая отправка офлайн-задач после восстановления подключения.

Важно: офлайн-режим не заменяет полноценную серверную синхронизацию. Изменения, сделанные без сети, становятся общими только после успешной отправки на сервер.

## Production-сборка

Соберите frontend:

```bash
npm run build
```

Собранные файлы будут находиться в `public/build`.

Типовой порядок деплоя:

```bash
composer install --no-dev --optimize-autoloader
npm ci
npm run build
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

Публичная директория веб-сервера должна указывать на:

```text
public
```

Для production-сервера также рекомендуется:

* включить HTTPS;
* настроить корректный `APP_URL`;
* ограничить доступ к `.env` и служебным файлам;
* настроить cron для Laravel scheduler, если появятся scheduled-задачи;
* настроить queue worker, если приложение начнёт использовать фоновые задания.

## Безопасность

В проекте используется упрощённая авторизация по персональной ссылке. Это удобно для домашнего приложения, но требует аккуратного обращения с токенами.

Рекомендации:

* не публикуйте magic-ссылки в открытом доступе;
* передавайте ссылки только приватно;
* используйте HTTPS на production-сервере;
* при подозрении на утечку пересоздайте пользователя или токен;
* не используйте этот механизм без доработок для публичного сервиса с большим количеством пользователей.
